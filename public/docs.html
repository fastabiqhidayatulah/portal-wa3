Dokumentasi API Portal WhatsApp

Panduan lengkap integrasi, rincian parameter, serta contoh kueri HTTP untuk mengirim pesan teks, berkas media, dan integrasi eksternal.

Skema Otentikasi

Portal menyediakan dua jenis endpoint dengan metode autentikasi yang berbeda sesuai dengan arsitektur sistem Anda:

  • Endpoint Internal (Dashboard): Terletak di bawah basis rute /api/internal. Menggunakan otentikasi sesi aktif (Cookie) yang otomatis disimpan ketika Anda melakukan login via dashboard utama.
  • Endpoint Eksternal (Aplikasi Mandiri): Menggunakan rute publik /api/external. Memerlukan Header Kunci API yang dilewatkan melalui properti: x-api-key sesuai variabel EXTERNAL_API_KEY di server.
POST /api/external/send-message (Personal JID)

Kasus Penggunaan: Mengirim Pesan Teks ke Nomor WhatsApp Personal

Mengirim pesan teks langsung ke satu nomor ponsel individual. Format nomor akan dinormalisasi secara otomatis (contoh: awalan 08xxx akan dikonversi menjadi 628xxx).

Payload JSON (Raw):

  • targetType (String - Wajib): Set nilainya menjadi "personal"
  • target (String - Wajib): Nomor telepon tujuan (contoh: "08123456789" atau "628123456789")
  • message (String - Wajib): Isi konten teks pesan Anda. Mendukung karakter baris baru (\n) dan emoji.

Contoh cURL Request:

curl -X POST http://localhost:3000/api/external/send-message \
  -H "Content-Type: application/json" \
  -H "x-api-key: [EXTERNAL_API_KEY_ANDA]" \
  -d '{
    "targetType": "personal",
    "target": "08123456789",
    "message": "Halo! Ini adalah pesan konfirmasi pesanan dari Portal WA Nusantara.\n\nStatus: Sukses"
  }'
POST /api/external/send-message (Grup)

Kasus Penggunaan: Mengirim Pesan Teks Ke Grup WhatsApp Berdasarkan Nama Grup

Mengirim pesan pengumuman atau notifikasi grup. Server Portal WA akan secara dinamis mencari grup yang sesuai dengan nama subjek grup yang Anda masukkan (tidak sensitif terhadap huruf besar/kecil).

Payload JSON (Raw):

  • targetType (String - Wajib): Set nilainya menjadi "group"
  • target (String - Wajib): Nama subjek grup Anda (contoh: "Grup Alumni Nusa" atau "Dev Team")
  • message (String - Wajib): Konten pesan teks pengumuman.

Contoh cURL Request:

curl -X POST http://localhost:3000/api/external/send-message \
  -H "Content-Type: application/json" \
  -H "x-api-key: [EXTERNAL_API_KEY_ANDA]" \
  -d '{
    "targetType": "group",
    "target": "Dev Team",
    "message": "[Pemberitahuan Sistem] Deploy server berhasil diselesaikan pada jam 12:00 WIB."
  }'
POST /api/internal/send-media (Attachment Lokal)

Kasus Penggunaan: Mengirim Berkas Dokumen, Gambar, Audio, atau Video yang Tersimpan di Komputer/Server Lokal

Mengunggah file biner langsung ke server portal, yang kemudian diteruskan secara instan oleh bot WhatsApp ke kontak tujuan. Mendukung multi-target personal dan grup secara bersamaan.

Payload Request (Format multipart/form-data):

  • file (File - Wajib): File biner media dari disk lokal Anda.
  • personalTargets (String - Opsional): Koma sebagai pemisah daftar nomor HP tujuan (contoh: "0812345,0898765").
  • groupId (String - Opsional): ID JID grup (contoh: "12036302@g.us") ATAU nama grup langsung (contoh: "Dev Team").
  • caption (String - Opsional): Caption untuk lampiran gambar atau video.

Contoh cURL Request (Upload Gambar/Dokumen):

curl -X POST http://localhost:3000/api/internal/send-media \
  -F "file=@/home/user/document/laporan_penjualan.pdf" \
  -F "personalTargets=081234567890" \
  -F "caption=Berikut lampiran laporan bulanan kami" \
  -b cookie.txt
POST /api/internal/send-media-url (Attachment URL)

Kasus Penggunaan: Mengirim Media Menggunakan Direct Link Internet

Mengirimkan gambar, video, musik, atau dokumen PDF yang sudah terunggah di hosting cloud publik. Server akan melakukan streaming berkas dan mengirimkannya sebagai objek media asli di obrolan chat WhatsApp target Anda.

Payload JSON (Raw):

  • url (String - Wajib): Tautan langsung berkas (direct link) di internet (contoh: "https://website.com/brosur.jpg").
  • personalTargets (String - Opsional): Koma sebagai pemisah daftar nomor HP tujuan.
  • groupId (String - Opsional): ID JID grup atau nama grup tujuan.
  • caption (String - Opsional): Teks penyerta berkas media.

Contoh cURL Request (Kirim Media via URL):

curl -X POST http://localhost:3000/api/internal/send-media-url \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://images.unsplash.com/photo-1579202673506-ca3ce28943ef?w=500",
    "personalTargets": "081234567890",
    "caption": "Selamat pagi! Silakan tinjau poster promosi terbaru kami."
  }' \
  -b cookie.txt

Tips Format Data & Batasan

  • Throttling Anti-Spam: Untuk mencegah nomor WhatsApp Anda diblokir, disarankan memberikan jeda (delay) minimal 3-5 detik antara setiap pemanggilan API jika mengirimkan pesan secara masal.
  • Identifikasi Otomatis Berkas: Saat menggunakan metode URL, server Portal WA secara cerdas mendeteksi format berkas melalui ekstensi nama file (.jpg, .png, .mp4, .pdf) untuk menentukan jenis payload yang dikirimkan.
  • Keamanan Sesi: Endpoint bertanda /api/internal/* memerlukan header cookie dari autentikasi login browser Anda, atau gunakan /api/external/* untuk diintegrasikan secara langsung dari sistem eksternal Anda.
